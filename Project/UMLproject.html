The system that Team 3 decided to use for this project is a Point of Sale System for Dockers Restaurant which is located in Fox Lake, Illinois.  Dockers Restaurant is owned by Mario and Marciano Martinez, two brothers who immigrated from Mexico in the 1990’s.  Dockers Restaurant generates the majority of its revenue in the months of May through October.  The peak month of sales is August where the restaurant generates a gross income of $500,000.  In these 6 months the gross income varies between $300,000-$500,000 per month.  The reason that Dockers Restaurant utilizes a Point of Sales System is to try and make the business more efficient.  Prior to the use of the Point of Sales System at Dockers each waiter/bartender had to manually submit the orders to the kitchen/bar while also calculating the total sales and proper tax amount for each order.  This process was inefficient and required more staff while also making customers wait a longer period of time in order to receive their bill.  This existing process resulted in a lower table turnover ratio which negativity impacted both the waiters/bartenders and owners.

The current system helps make the business more efficient than without the use of a Point of Sale System.  The Point of Sale system works by individually keeping track of each waiter’s/bartender’s amount of tables, payment method by the customer, total sales, and total tax amount.  This system automatically calculates the total sale and tax amount per each table which eliminates the need for each waiter/bartender to manually calculate this.  This system allows the manager at the end of each night to view the total amount of sales with the click of a button.  This is a convenience factor which saves the manager hours at the end of the night by eliminating the need to calculate final sales by hand.

 

 

 

Problem/Opportunity

 

 

 

 

 

The first problem that this project will solve is replacing the current computers with new computers and a updated operating system.  The current computers that Dockers Restaurant uses are from the early 2000’s which are very outdated, slow, and inefficient in terms of today’s technology.  The current operating system does not allow each waiter/bartender to switch between computers during the shift.  All of the waiters/bartenders are required to pick one computer and only use that computer.  If the waiters/bartenders switch to other computers this will cause the entire operating system to crash which makes all computers not usable until the entire system is reset.  This is very inefficient because at times during a shift some computers are not being utilized while other computers have multiple waiters/bartenders waiting to enter their customers’ orders.  A new current operating system will allow all waiters/bartenders to switch between computers.  The second problem that the waiters/bartenders face is uncalibrated screens do to years of wear and tear.  These uncalibrated screens make it difficult to properly enter an order which slows down other waiters/bartenders.  New computers will be able to support an up to date operating system while being more functional.

The second problem that this project will solve is changing the format and layout of the computer system.  The current system has too many unneeded buttons which confuses the waiters/bartenders which slows down the order entering process.  This new format and layout will reduce amount of options to select within the computer.  This will help reduce the time on the computer which will allow servers/bartenders to focus more time on the guests.

The final part of this project will be an opportunity for the restaurant.  Dockers Restaurant does not currently have a system for tracking inventory.  The owner Marciano has to manually count the inventory once a week.  As a team we came up with the idea of being able to track inventory by what the waiters/bartenders submitted.  This will be included within the end of the report each night which will allow Marciano to view orders submitted vs actual inventory.  This can help the owners be more aware to food waste which will help save the restaurant money.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Proposed System

 

 

 

 

 

The new Point of Sale System is new and improved and will help make Dockers Restaurant more efficient than ever before.  The new POS System will work by giving waiters/bartenders their own unique 4 digit pin password.  Once the waiters/bartenders log into their system they will be instructed to enter the new table number and person count for a specific customers table.  Once the table number and person count has been entered the table number will appear on the screen allowing the waiter/bartender to either enter a new customer table or access an existing customers already that was already created table.  When a waiter/bartender clicks on an existing table they will be given basic button options such as: Food, Beverage, Payment or Back.  When the waiter/bartender enters one of the previously stated button sections they will be taken to the next screen which will be ordered alphabetically.  For example, if the Food button was entered the waiter/bartender will see the different types of food categories.  If the waiter/bartender selected Food then selected Burgers, the entire burger menu would appear.  This layout will help make the order entering process more efficient.

The final development that our team came up with is an inventory tracking system.  The system will operate off of what waiters/bartenders submitted via their order.  It will work by storing all orders that the waiters/bartenders submitted and then calculate each individual menu category that was sold which will appear on the waiters/bartenders end of the night report.  The owners will have to decide on how much inventory should be going into preparing a specific order, in order to establish a metric.  Once this metric is established the owners can compare actual inventory numbers to what was actually submitted by the waiters/bartenders.  This will give the owners insight on where inventory is going.
